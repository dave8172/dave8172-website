## Overview

This project bridges the gap between the flexibility of spreadsheets and the robust data 
integrity of a <RandomColor>dedicated SaaS</RandomColor>. By embedding a **Vue.js SPA** directly within the Google 
Sheets sidebar, I’ve created a "Headless ERP" experience. 

Instead of manual cell entry, users interact with a reactive, type-safe interface that 
validates data in <RandomColor>real-time</RandomColor>, communicates with a backend REST API, and automates the 
entire bookkeeping lifecycle—from loading complex templates to securing license keys and 
API secrets.

## <RandomColor>Tech Stack</RandomColor>

* **Frontend:** Vue.js (Reactive UI/UX) & TypeScript (Type-safe logic)
* **Backend Integration:** REST API via Fetch/Axios
* **Runtime:** Google Apps Script (GAS) with `.gs` injection
* **Security:** Script Properties Service (for secrets/license keys)
* **Deployment:** Google Workspace Marketplace

## Strategic Value for Bookkeeping Businesses

For bookkeeping and accounting firms, manual data entry is the primary source of both overhead and errors. This application solves those pain points by treating the spreadsheet as a controlled database frontend.

### 1. Automated Template Loading
Bookkeepers often work with varied chart-of-accounts or client-specific formatting. This app allows users to fetch and **load financial templates dynamically** from a central API. With one click, the sheet is <RandomColor>perfectly structured</RandomColor>, pre-filled with the necessary headers, and ready for data.

### 2. High-Integrity Transaction Entry
Standard spreadsheets allow users to type anything anywhere. This SPA restricts inputs to **dynamically filled dropdowns** (fetched from your backend), ensuring that every transaction code, category, or client ID is <RandomColor>100% accurate</RandomColor> before it ever touches the sheet.

### 3. Secure Multi-Tenant Management
Managing multiple clients requires handling various API keys and license credentials. By utilizing **Google Script Properties**, this application stores sensitive "secrets" and license keys <RandomColor>securely</RandomColor> in the background. They are never exposed in the cells, protecting both the firm and the client.

### 4. Real-Time Success & Error Feedback
The interface provides immediate **status messages** (Success/Error/Warning) within the sidebar. When a transaction is pushed to the backend, the bookkeeper gets instant confirmation, while the sheet itself automatically updates with **conditional formatting** to highlight processed rows.
